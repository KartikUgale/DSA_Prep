### 1. Platform Independent : 
`.java (Source Code)` --> `Compiler (javac)` --> `.class (Byte Code)`--> `Interpreter/JIT compiler (JVM)` --> `OS`      

`JVM` find for `FileName.main()` and then code initiate from `main()` function.

>**.java (Source Code)**     
You write the Java program in a .java file.     

>**Compiler (javac) → .class (Bytecode)**        
The Java compiler (javac) compiles the `.java` file into a `.class` file, which contains bytecode (an intermediate, platform-independent code).    

>**Interpreter (JVM) → Execution**       
The Java Virtual Machine (JVM) loads the .class file and interprets the bytecode.
JVM translates `bytecode` into `machine code` using an interpreter or a `Just-In-Time (JIT) compiler`.    

>**Operating System (OS)**       
The machine code generated by the JVM runs on the underlying OS, executing the program.

### Abstract : 
Below example defines an abstract class Shape with an abstract method calculateArea() and a non-final instance variable b. The Rectangle class `extends` the Shape class and provides the implementation for the calculateArea() method.

        obj.b = 200;
        obj.calculateArea();

    public void calculateArea() {
        System.out.println("Area is " + (b * b));
    }

### MultiThreading in java: 
- In Java, a thread is a lightweight process that allows a program to operate more efficiently by doing multiple tasks at the same time.
- Threads can be used to perform complicated tasks in the background without interrupting the main program.
- There are two primary ways to create a thread in Java:    
  1. implementing the Runnable interface.
   
```java
class Process1 implements Runnable {
    public void run() {
        for(int i=1; i<=10; i++) {
            System.out.println("Process-1 i = "+i);
        }
    }
}

class Process2 implements Runnable {
    public void run() {
        for(int i=0; i<=10; i++) {
            System.out.println("Process-2 i = "+i);
        }
    }
}

public class MultiThread1 {
    public static void main(String[] args) {
        Process1 p1 = new Process1();
        Process2 p2 = new Process2();
        Thread t1 = new Thread(p1);
        Thread t2 = new Thread(p2);
        t1.start();
        t2.start();
    }
}
```
 2. extending the Thread class 

```java 
class Process1 extends Thread {
    public void run() {
        for(int i=1; i<=10; i++) {
            System.out.println("Process 1 : "+i);
        }
    }
}

class Process2 extends Thread {
    public void run() {
        for(int i=0; i<=10; i++) {
            System.out.println("Process 2 : "+i);
        }
    }
}

public class MultiThread2 {
    public static void main(String[] args) {
        Process1 p1 = new Process1();
        Process2 p2 = new Process2();
        p1.start();
        p2.start();
    }
}
```

### Exception Handling :
>Exception handling in Java is a mechanism that allows developers to **manage runtime errors effectively**, ensuring the program's normal flow is maintained despite unexpected events.  
Java provides a robust exception handling framework using keywords like try, catch, throw, throws, and finally.

- `try`: The "try" keyword specifies an exception block where code that might cause an exception is placed.
- `catch`: The "catch" keyword specifies the code block to be executed if an exception occurs in the try block.
- `finally`: The "finally" block will always be executed whether an exception occurs or not, ensuring that necessary clean-up code runs.
- `throw`: The "throw" keyword triggers an exception manually.
- `throws`: The "throws" keyword declares an exception that a method may throw.
  
>java throw - java catch     
java throw - `developer catch`  
developer throw - java catch    
developer throw - `developer catch`

### GUI (Graphical User Interface) :
For building graphical user interfaces (GUI) in Java, the most common frameworks are:
1. Swing (Lightweight, older but widely used)
    - javax.swing package
    - Components: JFrame, JPanel, JButton, JLabel, etc.
    - Event handling using ActionListener
2. JavaFX (Modern, preferred for new applications)
    - javafx.application.Application (main class).
    - Uses XML-based FXML for UI design
    - Supports CSS for styling
    - Can integrate with databases and web APIs
3. AWT (Abstract Window Toolkit) (Not used now)
    - AWT is Java’s oldest GUI framework, included in java.awt package. It provides native-looking UI components but is largely replaced by Swing and JavaFX because of its limitations.
    - Swing is built on top of AWT, and many Swing components extend AWT classes.

- JDBC - `Java Database Connectivity`.
- JDBC is a powefull API that allows java programs to access and manupulate data stored in a wide variety of databases.
- JDBC is an API that defines how a client can connect to a database, send SQL quires and statements, and retrive and manipulate the result.
- JDBC consists of a set of interfaces and classes written in java.

  1. JDBC-ODBC bridge driver
  2. Nativ API driver
